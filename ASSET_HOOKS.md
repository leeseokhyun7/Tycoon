# Asset Hooks

이 파일명으로 이미지를 넣으면 코드 수정 없이 우선 적용됩니다. 파일이 없으면 기존 기본 이미지가 그대로 보입니다.

이미지는 스프라이트 시트나 캐릭터 리스트가 아니라, 실제 화면에 들어갈 단일 오브젝트만 잘라낸 투명 PNG를 넣어야 합니다.

## 이미지

위치: `assets/skin/`

- `world.png`: 본게임 배경. 권장 960x1720 이상.
- `bbang-ready.png`: 완성/진열/주문 표시용 붕어빵. 투명 PNG 권장, 512x384 이상.
- `bbang-raw.png`: 반죽 상태 붕어빵. 투명 PNG 권장, 512x384 이상.
- `bbang-burnt.png`: 탄 붕어빵. 투명 PNG 권장, 512x384 이상.
- `customer.png`: 모든 손님 공통 대체 이미지. 투명 PNG 권장, 256x384 이상.
- `customer-0.png` ~ `customer-8.png`: 손님별 개별 대체 이미지. 있으면 `customer.png`보다 우선 적용됩니다.
- `ui-button.png`: 작은 버튼, 레시피 버튼, 상점 항목 버튼.
- `ui-panel.png`: HUD 패널, 진열대, 요약/팝업 패널.
- `ui-dock.png`: 하단 조작 영역.
- `ui-grill.png`: 열판 프레임.
- `ui-shop.png`: 상점/정산 패널. 있으면 `ui-panel.png`보다 우선 적용됩니다.

인트로 화면은 현재 `assets/new/intro.png`를 직접 교체하면 됩니다. CSS 훅으로 없는 파일을 강제하면 시작 화면이 깨질 수 있어서 안전하게 기존 경로 교체 방식으로 유지했습니다.

## 오디오

위치: `assets/sfx/`

- `bgm.mp3`: 배경음악. 없으면 기존 내장 생성 BGM이 재생됩니다.
- `tap.mp3`: 탄 붕어빵 치우기 같은 일반 탭.
- `cook.mp3`: 반죽 붓기.
- `fill.mp3`: 속 넣기.
- `flip.mp3`: 뒤집기.
- `collect.mp3`: 진열대로 수거.
- `serve.mp3`: 손님에게 일부 전달.
- `coin.mp3`: 주문 완료/보상.
- `burn.mp3`: 타버림.
- `fail.mp3`: 불가능한 조작/돈 부족.
- `buy.mp3`: 상점 구매.

브라우저는 `mp3`, `ogg`, `wav`를 재생할 수 있지만 현재 훅 파일명은 `.mp3`로 고정되어 있습니다. 다른 확장자를 쓰려면 파일명을 위 목록과 동일한 `.mp3`로 맞춰주세요.
